# Example 10: Export/Import

**What you'll build:** Backup and restore workflows for Fabrica-generated APIs using the generated export/import server subcommands (no running API required).

**What you'll learn:**
- Using generated export/import server subcommands (v0.4.0+)
- Direct storage access for efficient data portability
- Import modes: upsert, replace, skip
- Offline backups with transactional safety

**Difficulty:** ⭐⭐ Beginner-Intermediate
**Time:** 10-15 minutes

## Overview

Export/import functionality enables:
- **Backups**: Regular snapshots for disaster recovery
- **Migration**: Moving data between environments (dev → staging → prod)
- **Version Control**: Tracking resource definitions in Git
- **Seeding**: Populating test/demo environments with known data

## Implementation

### Generated Server Commands (v0.4.0+)

Fabrica v0.4.0+ generates export/import commands directly into each server binary. These commands work **offline** with direct storage access—no running server needed:

```bash
# Export all resources to YAML (no server required!)
./cmd/server/server export --format yaml --output ./backup

# Export specific resource types
./cmd/server/server export --kinds Server,Rack --output ./partial

# Import from backup
./cmd/server/server import --input ./backup

# Dry run to preview changes
./cmd/server/server import --input ./backup --dry-run

# Replace mode (delete all first)
./cmd/server/server import --input ./backup --mode replace
```

**Why this is the right approach:**
- **No server needed** - Works offline with direct database access
- **No network overhead** - Direct storage layer access
- **Type-safe** - Generated per-resource with proper struct handling
- **Transactional** - Atomic imports via `WithTx()` for consistency
- **Fast** - No HTTP round-trips
- **Works anywhere** - On your laptop, CI/CD pipelines, remote servers

**Architecture:**
- Commands generated into `cmd/server/export.go` and `cmd/server/import.go`
- Use `storage.Query{Resource}(ctx).All(ctx)` for direct storage access
- Support JSON and YAML formats (`--format`)
- Atomic operations via `storage.WithTx()` for safe imports
- Import modes: `upsert` (default), `replace`, `skip`
- Per-type organization with `--per-type` flag

There is no longer a REST-script fallback in this example. Use the generated commands above for all backup/restore workflows.

## Quick Start

1. **Generate a project with export/import support**

   ```bash
   fabrica init demo-export --storage-type ent
   cd demo-export
   fabrica add resource Sensor
   fabrica generate
   go mod tidy
   ```

2. **Start the server and create some test resources**

   ```bash
   # Terminal 1: Start server
   go build -o server ./cmd/server
   DATABASE_URL="file:data/demo.db?_fk=1" ./server serve &

   # Terminal 2: Create test resources
   curl -X POST http://localhost:8080/api/v1/sensors \
     -H "Content-Type: application/json" \
     -d '{
       "apiVersion": "example.fabrica.dev/v1",
       "kind": "Sensor",
       "metadata": {"name": "temp-sensor-01"},
       "spec": {"type": "temperature", "location": "rack-1"}
     }'

   # Create a few more...
   curl -X POST http://localhost:8080/api/v1/sensors \
     -H "Content-Type: application/json" \
     -d '{
       "apiVersion": "example.fabrica.dev/v1",
       "kind": "Sensor",
       "metadata": {"name": "temp-sensor-02"},
       "spec": {"type": "temperature", "location": "rack-2"}
     }'
   ```

3. **Export resources (can stop server before exporting)**

   ```bash
   # Optionally stop the server; export works offline
   ./server export --format yaml --output ./backup

   # Or export as JSON
   ./server export --format json --output ./backup-json
   ```

4. **Inspect exported data**

   ```bash
   tree backup/
   cat backup/sensors/sensor-*.yaml
   ```

5. **Import resources (offline)**

   ```bash
   # Dry run to preview
   ./server import --input ./backup --dry-run

   # Actually import
   ./server import --input ./backup
   ```

6. **Verify import worked**

   ```bash
   # Start server again and check resources were imported
   DATABASE_URL="file:data/demo.db?_fk=1" ./server serve &
   curl http://localhost:8080/api/v1/sensors | jq .
   ```

## Notes
- SQLite requires `_fk=1` in the DSN for foreign keys.
- Export/import commands are generated per project; use them instead of REST scripts.
- Works with any Ent-backed project generated by Fabrica v0.4.0+.
