# Manual verification: TokenSmith AuthN/AuthZ (generated service)

This checklist is intended for reviewers validating that **Fabrica-generated services** wire **TokenSmith** for authentication (AuthN) and authorization (AuthZ) without any Fabrica-local security engine.

No extra “hack servers” or runtime tooling should be generated by Fabrica beyond the normal service plus its dependencies (TokenSmith/Casbin when enabled).

## Preconditions

- You have a JWKS provider URL (for local testing a running IdP that exposes a JWKS endpoint).
- You have a generated service where security is enabled:
  - `Security.AuthN.Enabled=true`
  - `Security.AuthZ.Enabled=true`
  - `Security.AuthZ.Mode=enforce|shadow`
- You have a minimal policy available to TokenSmith (exact mechanism depends on TokenSmith configuration in the generated service; commonly an env var pointing to a local policy file).

## Environment

Set the required TokenSmith environment variables used by the generated service:

```bash
export TOKENSMITH_JWKS_URL="https://<your-idp>/.well-known/jwks.json"

# Select behavior:
#   enforce: deny on policy failure (403)
#   shadow: allow but log a structured deny decision
export TOKENSMITH_AUTHZ_MODE="enforce"   # or "shadow"

# Any additional TokenSmith/Casbin/policy loader env vars your deployment uses
# (e.g. a local policy file path).
```

Start the generated service.

## Checklist

### 1) Public endpoints bypass security

- `GET /healthz` returns **2xx** without an `Authorization` header
- `GET /readyz` returns **2xx** without an `Authorization` header
- `GET /metrics` returns **2xx** without an `Authorization` header (if enabled in the generated service)

### 2) Protected endpoint requires AuthN

Pick any generated resource endpoint mounted under the protected route group (for example: `GET /v1/<resource>`).

- Without token:
  - request returns **401** (or other non-2xx indicating missing/invalid authentication)

Also verify:
- An `OPTIONS` preflight request to a protected route is **not blocked** (should return 2xx/204 as configured).

### 3) AuthZ decisions: enforce vs shadow

Use a token that is valid for AuthN but **does not satisfy** the authorization policy for the protected endpoint.

#### Enforce mode

- `TOKENSMITH_AUTHZ_MODE=enforce`
- Request to protected endpoint returns **403** on policy deny

#### Shadow mode

- `TOKENSMITH_AUTHZ_MODE=shadow`
- Request to protected endpoint is **allowed** (2xx/normal handler result), but a **structured log/event** indicates an AuthZ deny decision was made.
  - Confirm logs contain a clear “deny” decision with route/method context.

## Non-regression / baggage check

- Confirm the generated service’s `go.mod` includes TokenSmith (and related Casbin deps) **only when** security is enabled.
- Confirm no additional “local auth servers”, “mock IdPs”, or other runtime tooling were added to generated output.
